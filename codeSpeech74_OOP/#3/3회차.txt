#1 HTML PARSER

(1) 
알고리즘: 변수의 상태와 흐름을 제어하는것

(2) 함수를 만들어야 할 때
- 코드가 중복되었을 때
- 코드가 별도의 독립행위가 있을때

(3) 함수의 중요한 역할
- 코드상의 알고리즘을 제거하고 어휘로 바꿔주는 행위

(4) 코드의 가독성이 높아지는 방법
- 알고리즘의 역할을 단어로 표현한다.
- text = '';
  text = textNode(text, 대상);

- 알고리즘은 인간에게 어렵기 때문에 함수를 만들어서 단어로 표현해야한다.
- 알고리즘이 커져서 특정역할을 할 때, 함수의 이름이 된다.
- 역할에 맞는 이름을 붙일때, 가독성은 높아진다.
